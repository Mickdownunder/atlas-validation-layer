# ATLAS Command Map v1

**Sandbox tools note:** When this document refers to **sandbox tools** such as numpy, scipy, or a Python runtime, it means the **research sandbox** used by Council and Experiment flows: the Docker image `operator-research-sandbox` with Python 3.11, numpy, and scipy (`operator/docker/research-sandbox/`, `tools/research_sandbox.py`). ATLAS itself uses the OpenClaw sandbox policy (`exec`, `read`, `write`) and the commands listed here. There is no Python/numpy runtime inside the ATLAS container.

## Role: execute and prove

The pipeline is designed like this: **June researches** across the system and produces a **thesis**. That thesis should then be **validated with code**. June delegates the request to ARGUS, and **ATLAS must execute and prove it**. In practice, the code that tests the thesis should run inside an ATLAS sandbox, and ATLAS returns the resulting evidence or counter-evidence.

**Safe environment for validation code:** ATLAS can execute thesis-validation code inside the **research sandbox** (Docker: no network, numpy/scipy available, enforced timeout). Command: `atlas-run-sandbox <path_to_script.py> [timeout_seconds]`. Logs: `logs/sandbox/<timestamp>-run/`. The code must be provided as a Python file, typically generated by June or ARGUS. Only numpy, scipy, and the Python standard library are available. No torch. No network.

## Approved macros

- `bin/atlas-sandbox-run <status|research|full|mini> [request text]` — Policy, Health, Mini-Audit usw.
- `bin/atlas-run-sandbox <path_to_script.py> [timeout_seconds]` — Validierungscode in Sandbox ausführen (max 30s default).
